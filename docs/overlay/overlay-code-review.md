# Код-ревью: Overlay System

Архитектурный анализ системы оверлеев fibo-ui CDK с сравнением популярных дизайн-систем.

## Оценка архитектуры

Система оверлеев **хорошо спроектирована и современна** — полностью основана на сигналах, реактивна, с композируемыми поведениями и чистым управлением жизненным циклом. API `createOverlay()` + коллбэк настройки `OverlaySession` — элегантнее и эргономичнее императивного подхода Angular CDK.

### Сильные стороны

- Управление состоянием через сигналы повсеместно (опережает Spartan, на уровне современных паттернов)
- Композируемые поведения закрытия (`closeOnFocusLeave`, `closeOnOutsideClick`, `closeOnBackdropClick`)
- Чёткая стратификация z-index по категориям с разумными значениями по умолчанию
- Мощный API `OverlaySession` (хуки жизненного цикла, управление эффектами, очистка)
- `isOverlayInBranch()` корректно обрабатывает иерархии вложенных оверлеев
- Очистка с учётом анимаций через слушатель `animationend`
- Встроенное восстановление фокуса через `restoreTriggerFocusOnClose()`

## Обнаруженные проблемы

### 1. Нет системы стратегий скролла

Система имеет только `overflow: hidden` на `<html>` для диалогов/подтверждений. Angular CDK предоставляет 4 стратегии: `block`, `reposition`, `close`, `noop`. Отсутствует:

- **Закрытие при скролле** — нет способа автоматически закрыть поповер, когда пользователь скроллит от триггера
- **Настройка на уровне оверлея** — стратегия жёстко привязана к категории, а не настраивается для каждого экземпляра

> Примечание: репозиционирование при скролле частично покрывается через `autoUpdate` из `@floating-ui/dom`.

### 2. Нет захвата фокуса (Focus Trap)

Диалоги не имеют реализации захвата фокуса. Когда диалог открыт, Tab может переместить фокус на фоновый контент. Angular CDK имеет `FocusTrap` / `FocusTrapFactory`, Spartan наследует это. Это **пробел в доступности** — WCAG требует удержания фокуса внутри модальных диалогов.

### 3. Нет управления ARIA-атрибутами

Контейнер оверлеев не управляет:

- `aria-modal="true"` на диалогах
- `aria-hidden="true"` на фоновом контенте (inert)
- `role="dialog"` / `role="alertdialog"`
- связкой `aria-labelledby` / `aria-describedby`

Spartan обрабатывает всё это через свою директиву Dialog. Компоненты fibo-ui могут обрабатывать это индивидуально, но это должно быть систематизировано на уровне CDK.

### 4. Нет атрибута `inert` на фоновом контенте

Когда модальный диалог открыт, фоновый контент должен быть помечен атрибутом `inert`, чтобы предотвратить доступ скринридера. Angular CDK обрабатывает это через `InteractivityChecker` + `FocusTrap`. Блокировка скролла (`overflow: hidden`) предотвращает визуальный скролл, но не выход фокуса/скринридера.

### 5. Единый глобальный обработчик Escape

`(document:keydown.escape)` на контейнере вызывает `closeTopmost()`. Это работает, но:

- Не передаёт событие в оверлей для кастомной обработки
- Нельзя предотвратить/кастомизировать для конкретного оверлея (например, «Вы уверены, что хотите закрыть?»)
- Spartan имеет input `disableClose`; CDK имеет `OverlayKeyboardDispatcher` с делегированием каждому оверлею

### 6. Нет поддержки Shadow DOM при клике снаружи

`closeOnOutsideClick` использует проверки `event.target`, но не учитывает границы Shadow DOM. Angular CDK использует `containsPierceShadowDom()` для обхода composed path. Если компоненты когда-либо будут использоваться внутри Shadow DOM, определение клика снаружи сломается.

### 7. Нет сохранения позиции скролла при блокировке

Блокировка скролла добавляет `overflow: hidden` к `<html>`, но не сохраняет позицию скролла с отрицательными смещениями. Это вызывает видимый скачок наверх при открытии диалогов. `BlockScrollStrategy` в Angular CDK кэширует позицию скролла и применяет `top: -${scrollY}px; left: -${scrollX}px` для предотвращения сдвига лейаута.

### 8. `*ngTemplateOutlet` вместо нативного потока управления

В `overlay-container.html` используется `*ngTemplateOutlet` (структурная директива). CLAUDE.md проекта предписывает использовать нативный поток управления (`@if`, `@for`). Хотя для `ngTemplateOutlet` нет нативного эквивалента, стоит отметить это для осведомлённости о консистентности.

## Как работают оверлеи в популярных дизайн-системах

### Angular CDK

- **Контейнер:** Единый сервис `OverlayContainer`, создаёт `div.cdk-overlay-container` в `document.body`
- **Портал:** Система `Portal` + `PortalOutlet` — `ComponentPortal`, `TemplatePortal`, `DomPortal`
- **Позиционирование:** `FlexibleConnectedPositionStrategy` — точка на элементе-источнике соединяется с точкой оверлея, поддерживает fallback-позиции
- **Скролл:** 4 стратегии — `block` (блокировка), `reposition` (перепозиционирование), `close` (закрытие), `noop`
- **Бэкдроп:** Анимированный `div.cdk-overlay-backdrop` с transition на opacity, fallback-таймаут 500мс
- **Клавиатура:** Глобальный `OverlayKeyboardDispatcher` с предикатами событий
- **Клики снаружи:** Глобальный `OverlayOutsideClickDispatcher`, учитывает Shadow DOM
- **Z-index:** Единый stacking context на контейнере (z-index: 1000)
- **Фокус:** `FocusTrap` в модуле a11y

### Spartan/Brain

- **Архитектура:** Директивно-ориентированная, на сигналах
- **Диалог:** Базовая реализация с богатым позиционированием и управлением состоянием
- **Поповер:** Расширяет паттерн Dialog — убирает бэкдроп, добавляет `align`, `sideOffset`, `offsetX`
- **Позиционирование:** Через CDK `OverlayPositionBuilder`, динамическая смена стратегий в рантайме
- **Состояние:** Все входные данные реактивны через `input()` + linked signals
- **Доступность:** Полная поддержка ARIA через Dialog (`ariaLabel`, `ariaLabelledBy`, `ariaDescribedBy`, `ariaModal`)
- **Ключевая особенность:** `disableClose` — опция запрета закрытия

### Taiga-UI

- **Паттерн:** Полиморфная система контента через `@taiga-ui/polymorpheus`
- **Портал:** Абстрактный класс `TuiPortal<T>`, требует хост-компонент `tui-root`
- **Открытие:** Метод `open()` возвращает `Observable` с автоочисткой при отписке
- **Контекст:** Создаёт изолированный инжектор с `POLYMORPHEUS_CONTEXT`
- **Ключевая особенность:** Observable-паттерн для операций с порталами

### Zart

- **Подход:** Тонкая обёртка над Angular CDK Overlay
- **Триггеры:** `click`, `hover`, `null` (программное управление)
- **Позиции:** Статическая карта позиций (top/bottom/left/right) с фиксированными смещениями
- **Клики снаружи:** Через `overlayRef.outsidePointerEvents()` от CDK

## Ключевые архитектурные паттерны индустрии

### 1. Паттерн Portal/Content Projection (стандарт CDK)

Классы Portal инкапсулируют «что рендерить», интерфейс PortalOutlet определяет «где рендерить». Разделение позволяет гибкую композицию.

### 2. Паттерн стратегии позиционирования

Интерфейс стратегии с методами `attach()`, `apply()`, `dispose()`, `detach()`. Откреплён от оверлея, может заменяться в рантайме.

### 3. Паттерн стратегии скролла

Единообразный интерфейс: `enable()`, `disable()`, `attach()`. Несколько реализаций для разных сценариев. Привязан к жизненному циклу оверлея.

### 4. Паттерн глобального диспетчера событий (CDK)

Единый глобальный слушатель для кликов/клавиатуры на документе. Поддерживает список присоединённых оверлеев. Диспетчеризует события к верхнему оверлею с проверкой предиката. Эффективен для многих оверлеев.

### 5. Паттерн реактивности на сигналах (Spartan)

Все входные данные реактивны. Эффекты автоматически реагируют на изменения. Обновления стратегии позиционирования через computed-эффекты.

### 6. Паттерн инъекции контекста (Taiga-UI)

Портал создаёт изолированный инжектор с контекстом. Гибкое разрешение типов контента.

## Рекомендации (по приоритету)

1. **Захват фокуса (Focus Trap)** — реализовать директиву `FocusTrap` или использовать CDK-шную. Автоматически применять к диалоговым оверлеям.
2. **`inert` на фоне** — когда модальный диалог открыт, устанавливать атрибут `inert` на корневом элементе приложения (исключая контейнер оверлеев).
3. **Сохранение позиции скролла** — кэшировать `scrollX/Y` и применять отрицательные смещения при добавлении `overflow: hidden`.
4. **Систематизация ARIA** — добавить поддержку `role`, `aria-modal`, `aria-labelledby` на уровне CDK, а не только в отдельных компонентах.
5. **Настраиваемая обработка Escape** — разрешить оверлеям перехватывать/предотвращать закрытие по Escape (например, коллбэк `canClose` на `OverlaySession`).
6. **Поведение закрытия при скролле** — добавить `closeOnScroll()` в `overlay-behaviors.ts`.
